# 役割

4つの役割。1人運用は中2つを多用し、上を畳める。大組織は各々に人/セッションを割く。

## リーダー（オーケストレーション）

把握と振り分けのために開くワークスペース。案件作業を**積み上げる場所ではない**。

- **地図**（`INDEX`）を持つ：何が存在し、どこにあり、どんな状態か。
- 依頼を正しい作業員へ振る。横断/軽作業は自分でやることもある。
- このディレクトリの memory は**名簿＋横断的な決定**だけ——個別案件の詳細は持たない。

柔軟さの注記：リーダーは軽い/横断的な作業をその場でやって*よい*。規律は、
**生んだ案件固有の知識を、その作業員の memory へ振り分ける**こと。リーダーに留めない。

## 秘書（問い合わせ窓口）

「どこに何が／設定は／状態は／この用語は何か」に答える1か所。

- **インフラ＆参照事実**を持つ：パス・アカウント・設定値・常設の参照・決議録・用語集。
- 作業する場所ではなく、相談される場所。作業員とリーダーが尋ね、各自で再導出しない。
- reference/infra 知識の**正しい家**——そうした知識は案件 memory に散らさず、ここに住む。

## 作業員（実行者）

各案件/領域が専用ディレクトリを持つ。ここで実作業が起きる。

- 自分の1案件の実作業をする。
- **自分の案件の memory だけ**持つ——他案件は持たない。
- 状態が変わったら、リーダーの `INDEX` へ1行で反映する。
- 参照（パス・設定・用語）は、infra を抱え込まず秘書に尋ねる。

## 経営 / 領域リード（任意・大組織向け）

1領域の複数作業員の上に、成果を担い・優先度を決め・上へ報告するリードを置ける
（`templates/reports/monthly-report.md`）。**1人運用は通常これをリーダーに畳む。**
別個の説明責任が本当に必要な複数領域があるときだけ導入する。

## これを安くする唯一の仕組み

ランタイムは指示と memory を**ディレクトリごと**にスコープするので、役割は概ね*配置*：
役割宣言（`CLAUDE.md`）と、正しい memory が正しいフォルダに住むこと。
あるフォルダでエージェントを開くこと*が*、その役割に入ることです。
