# 安全

システムが壊れずに自分を変えるための規則。あらゆる構造変更（秘書の設置・memory 移行・denoise・新ピース採用）に適用。

## 譲れない物

1. **変更前にバックアップ。** 対象をタイムスタンプ付きバックアップへコピーし、*成功を確認*してから破壊的/移動操作を行う。
2. **スコープ内に留まる。** その手順が名指す物だけ触る。他のディレクトリ/ファイルへさまよわない。
3. **追加と移動。削除しない。** 追加的変更と移設を、削除より優先する。削除するなら、バックアップから可逆であること。
4. **資産を守る。** 働き方/feedback の memory や既存ツールを潰さない。それらは機能。
5. **迷ったら残す。** 曖昧さは無変更を既定とする——*不可逆な操作については*。可逆な物には決断的でよい（下記）。
6. **何を変え、どう戻すかを報告。** 毎回、1行で。

## 安全＝可逆性であって麻痺ではない

「決して動かないこと」で安全にしたシステムは何も達成しない。解：undo を安くし、明確なケースで動く。

- **1コマンド復元**（バックアップ時に作る小さな `restore.sh`）が、戻すのを簡単にする。
- **粒度復元**：バックアップは丸ごとコピーなので、全部を戻さず1ファイルだけ引き戻せる。やりすぎは全部やり直しではなく `cp` 1回で済む。
- 安い undo があれば、「迷ったら残す」は*本当に*曖昧な物に適用され、明確に死/明確に正しいケースは決断的に処理される。慎重さを「何もしない」に潰させない。

## 復元ヘルパー（パターン）

バックアップ時に、戻すのが1コマンドで確実になるよう小さな復元スクリプトも書く：

```bash
#!/usr/bin/env bash
BAK="<path>/<target>.bak-<date>"
TGT="<path>/<target>"
[ -d "$BAK" ] || { echo "no backup; aborting"; exit 1; }
rm -rf "$TGT" && cp -r "$BAK" "$TGT" && echo "restored"
```

## 自己改変の注意

エージェントが*自分の* memory や設定を編集するとき、それは最も事情を知る判断者であり**同時に**最も静かにドリフトしうる者。
レールを掛け続ける：バックアップ・スコープ・資産保護・報告＋復元。エージェントの自信は安全装置ではない——バックアップが安全装置です。
