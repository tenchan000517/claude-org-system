# 判断フレーム

決定がどう下され、記録されるか。役割間で一貫し、蒸し返されないように。

## 決めたら*理由*を記録する

理由のない決定は、新しい人が見るたびに再議論される。あらゆる非自明な決定は、
**何を決めたか＋なぜ＋日付**として捕捉する（decisions KB と決議ログ参照）。理由が恒久部分です。

## 判断軸（領域ごと）

各領域は、繰り返し量る問いの短い明示リスト——その**判断軸**——を持つと良い。
毎回基準を再導出せず、状況をその軸に当てて採点する。適応用の汎用軸：

1. **スコープ** — これは自分が担う範囲内か、別の取り組みか？
2. **可逆性** — 安く戻せるか？（戻せるなら決断的に。戻せないなら減速。）
3. **コスト vs 効果** — 労力/文脈コストが価値に見合うか？
4. **資産への影響** — 守るべき物（働き方ルール・ツール・信頼）を脅かすか？
5. **確信度** — 実際に*知って*いるか、決めつけか？（決めつけなら先に検証。）
6. **境界** — 守るべき機密/権限の境界を越えるか？

領域は独自の軸項目を足す（顧客向け領域なら「顧客要望との適合」、品質ゲート領域なら「品質基準を通るか」）。
実際に使える程度に短く保つ。

## 「決断的 vs 慎重」の規則

- **可逆**な操作＋明確なケース → 決断的に動く（可逆性が安全網）。
- **不可逆**な操作、または真に曖昧 → 減速し、確認し、無変更を既定とする。

これが「安全に」を「何もしない」に潰させず、「決断的に」を無謀にさせない。分かれ目は*可逆性*であって勘ではない。

## 失敗・撤退の判断

成果を担うことには、うまくいっていないときにそれを宣言する義務が含まれる。継続的な未達を見たリードは、
負けている取り組みを惰性で続けさせず、自ら方向転換や停止を提案する。撤退（とその理由）の記録は、開始の記録と同じく重要です。
