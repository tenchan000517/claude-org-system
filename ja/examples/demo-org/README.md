# 実演：複数案件を抱える1人運用

システム全体を配線した**架空**の例。名前や詳細は作り物——自分の物に置き換えて。

## 状況（before）

「Sam」は1人スタジオで、クライアントの動画編集・マーケサイト構築・月次レポート・雑務をやっている。
以前は全部1つの `~/work` フォルダで。memory には ~40 の混在メモがあり、「クライアントXのメールサーバは？」を
探すのにスクロールが要った。セッションは遅く、Sam は時々古いメモに基づいて動いていた。

## after：構造

```
~/work/                      <- リーダー（ここを開いて把握＆振り分け）
  CLAUDE.md                  リーダー宣言
  INDEX.md                   地図：4案件・状態・次の一手
  secretary/                 秘書（問い合わせ窓口）
    KB.md                    メール設定・素材パス・アカウント・用語集
    rag-video-pipeline.md    反復する編集手順＋落とし穴
~/work-client-edit/          <- 作業員（編集はここを開く）
  CLAUDE.md                  作業員宣言
  HANDOFF.md
~/work-marketing-site/       <- 作業員
  CLAUDE.md
~/work-monthly-report/       <- 作業員
  CLAUDE.md
```

## 1日の回り方

1. Sam が `~/work` でエージェントを開く → `INDEX.md` を読む → クライアント編集が「進行中・次：セクション3をカット」と分かる。
2. `~/work-client-edit/` でエージェントを開く → その `HANDOFF.md` を読む → カットする。新しいメモは*この*作業員の memory に落ちる（リーダーではない）。
3. 編集中「標準化したコーデックは？」→ 推測せず**秘書**（`secretary/KB.md`）に尋ねる。
4. 1日の終わり → `HANDOFF.md` を更新（次：音声パス）＋リーダーの `INDEX.md` に1行の状態。

## どう段階導入したか（GETTING-STARTED どおり）

- **ステップ1：** `INDEX.md` ＋リーダー宣言を作る（地図だけ；他は動かさない）。痛みが即下がった。
- **ステップ2：** 各案件を自分のフォルダで開き始める。新しい memory が正しい家に落ち始めた。
- **ステップ3（1週間後、「あの設定どこ」が繰り返し刺さってから）：** `secretary/KB.md` を索引として立て、その後 reference メモを段階的にそこへ*移動*（先にバックアップ）して `~/work` を痩せさせた。
- 省いた：月次の経営報告レイヤ（1人なら INDEX が状態を映す）、伴走ツール（任意のまま）。

## 結果

どのフォルダを開いてもそのスコープだけ読まれる。「どこに何が」は一目で引ける。リーダーの memory は名簿だけ。
大事な物は消えていない——reference メモは*家へ移設*され、各ステップに1行の undo があった。
