# 実演：多部署組織

システムを組織規模で配線した**架空**の例。すべて作り物——自分の物に置き換えて。（1人版は `../demo-org/`。）

## 状況

「Northwind」は複数の異なる領域を持つ小スタジオ。以前は全部が1ワークスペースと1つの混在 memory に住み、
「誰が何を担うか」「数字に当たっているか」を綺麗に言えなかった。

## 構造（システム全体を配線）

```
~/northwind/                    <- 組織リーダー（ここを開いて部署横断で把握＆調整）
  CREDO.md                      中央クレド — 各部署がここから派生
  INDEX.md                      組織地図：部署・リード・領域・状態
  secretary/                    共有インフラ＆情報窓口（全部署が尋ねる）
    KB.md  kb-yaml/  rag-*.md  ingest/
  governance/
    resolution-log.md           決着した方向性レベルの決定
    strategy-2026-01.md         ある戦略レビューのメモ
  reports/
    content-2026-01.md          ある部署の月次報告

~/northwind-content/            <- 部署（profit/impact センター）
  CLAUDE.md                     部署働き方ドキュメント（役割・KGI/KPI・範囲）— CREDO から派生
  HANDOFF.md
  <案件ごとの作業員 dir…>

~/northwind-platform/           <- 部署（cost/enabling センター）
  CLAUDE.md                     （「看板数字は動かさない；動かす側を支える」）
  HANDOFF.md

~/northwind-clients/            <- 部署（profit/impact センター）
  CLAUDE.md
  HANDOFF.md
```

## どう噛み合うか

- **CREDO** が幹。各部署の `CLAUDE.md` は*その*役割・センター種別・目標を述べ、行動原則は再記述せず**継承**する。
- **組織 INDEX** が部署＋リード＋状態を載せる——リーダーの地図を1段上で。
- **秘書**は共有：全部署がパス・アカウント・設定・用語をそこに尋ね、誰もインフラを再導出しない。
- **ガバナンス**：方向性決定は決議として記録され、再決定まで部署を拘束する；定期の戦略レビューがそれを生む。
- **説明責任**：各 profit/impact 部署は KGI を担い、enabling 部署は動かせない看板数字でなく「何を可能にしたか」で評価される。リードは月次で固定フォーマット報告。

## どう導入したか

1人版と同じ段階の道を1段上で：まず**組織地図＋CREDO**を立て；各部署に専用ディレクトリ＋働き方ドキュメントを与え；
**秘書**を共有し；複数の説明責任者が調整を要したとき**ガバナンス**（決議ログ）を足し；時間比較が要り始めたとき**報告**を足した。
どれも、その価値が認められる前には足さなかった。

## 縮小の注記

1人運用はこれを畳む：経営・リード・リーダーは全部あなた；部署は案件フォルダになり；ガバナンスは秘書内の決定リスト。
*形*は同一——人数が変わるだけ。`../demo-org/` 参照。
