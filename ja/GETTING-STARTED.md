# はじめに

最小導入の道筋。**システム全体を一度に入れない。** 地図を立て、実際に痛みが出たときだけピースを足す。

## 始める前に：安全規律（常時オン）

1. 置き換え/移動の前にバックアップ（`cp -r 対象 対象.bak-<日付>`）。
2. その手順のスコープだけを触る。他は触らない。
3. 追加と移動。削除しない。働き方ルールの memory や既存ツールを潰さない。
4. 迷ったら残す。
5. 各手順の後、**何を変えたか**と**1行の戻し方**を報告する。

## ステップ1 — 地図（固定の第一手・必ずやる唯一の手順）

これだけが非任意で、純粋に追加のみ（memory はまだ触らない）。

1. catch-all ワークスペースで「今ライブな物」を洗い出す。
2. ルートに `INDEX` を作る：*案件 / 場所 / 状態 / 次の一手* の表。
3. ルートにリーダー宣言（`CLAUDE.md` 系ファイル）：「ここはリーダー、地図を持ち作業員へ振る、案件作業を抱えない」。

効果：「どこに何が」が一目で引ける。戻し方：新規2ファイルを消す。

## ステップ2 — 案件ごとのディレクトリで動き始める（無料・本当の切替）

以降、**案件の作業はその案件のフォルダでエージェントを開く**。リーダー/INDEX は振り分けにだけ使う。
新しい memory が自動で正しい家に溜まり始める。コストゼロで、最も効く習慣。

## ステップ3以降 — 実際の痛みが出たときだけ1ピース

感じている痛みに合うピースを選ぶ。一度に1つ。

| 感じている痛み | 足すピース |
|---|---|
| 「設定/パス/アカウントを毎回探してる」 | **秘書**を立てる（`secretary/KB.md`） |
| 「ある案件のメモが他を埋もれさせてる」 | その案件 memory を作業員 dir へ移動（先にバックアップ） |
| 「働き方ルールが各所に散ってる」 | 普遍ルールをランタイムの**グローバル**設定へ昇格 |

秘書については、**索引**から始め（ポインタ＋要点・現物は残す）、その後 infra/参照メモを段階的に
秘書の家へ**移動**（バックアップ・消さず移す）して catch-all を実際に痩せさせる。
「reference を守る」は *消すな* であって *正しい家へ移すな* ではない。

## 空振り防止

「痛んだときだけ足す」だけだと、結局何も入れずに止まる。だから**ステップ1は固定**：常に地図を先に作る。
その後は実際の摩擦に駆動させる。

## 任意レイヤ（別途判断）

伴走ツール（マスコット・補助エージェント・追加自動化）は**コアではない**。
コア（リーダー/秘書/作業員）はそれ無しで機能する。任意ツールは後で、単体の価値で、
既存の物との競合を見ながら採る。
