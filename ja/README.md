# claude-org-system

*English: [`../README.md`](../README.md) ／ 日本語: 本ページ*

AI コーディングエージェント（Claude Code 等）のための、**最小で再利用可能な「組織オペレーティングシステム」**。
過負荷になった単一ワークスペースを **リーダー → 秘書 → 作業員** の構造に変え、文脈をスコープ内に保ち、
「どこに何があるか」が常に引ける状態にし、システムが静かに壊れないようにします。

## 解決する問題

1つのワークスペース（1フォルダ）が *すべて* を担う——オーケストレーションも、無関係な多数の案件の実作業も——
と、3つが腐ります。

1. **memory が山になる。** 全案件のメモが1か所に積もり、何も見つからない。
2. **ルータがない。** 「何が存在し、どこにあり、どんな状態か」の地図がない。
3. **ドリフトと混乱。** 肥大した混在文脈はエージェントを遅くし、誤ったものに手を出させる。

## 解決の形

3つの役割。AI エージェントが既に持つ「ディレクトリごとに文脈をスコープする」性質に乗せます。

| 役割 | 何か | 仕事 |
|---|---|---|
| **リーダー** | オーケストレーション用フォルダ | 地図を持ち、作業員へ振る。案件作業を抱え込まない |
| **秘書** | 1つの問い合わせ窓口（KB） | 「どこに何が／設定は／状態は」に答える |
| **作業員** | 各案件の専用フォルダ | 実作業をする。自分の案件の memory だけ持つ |

核となる仕組み：**多くのランタイムは作業ディレクトリごとに指示と memory をスコープする。**
だから *各案件をその専用ディレクトリで作業する* だけで、山は自動的に分かれます。

### コア vs 完全版

**コア**（役割 01–05 ＋ 基本テンプレ）だけで、1人運用や単一チームには十分で、単独で機能します。
**組織レイヤ**（06–08：判断フレーム・部署・ガバナンス）と重いテンプレ（部署クレド・決議ログ・月次）は
多領域組織向け。**身の丈に合う分だけ採る**——コアは組織レイヤに依存しません。
