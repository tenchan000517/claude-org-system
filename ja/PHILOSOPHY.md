# 思想

システム全体が立つ原則。他のすべて（役割・雛形・手順）は、これらを具体化しただけのものです。

## 1. オーケストレーションと作業を分ける

「何があるかを知り振り分ける場所」（リーダー）が、「実作業を抱え込む場所」を兼ねてはいけない。
両者が混ざると、ワークスペースは catch-all になり、memory は検索不能の山になる。地図と労働は別の部屋に。

## 2. 文脈をディレクトリでスコープする

現代の AI エージェントは指示と memory を**作業ディレクトリごと**に読み込む。これが最も安いレバー：
各案件をその専用ディレクトリで作業すれば、文脈は自動的に分かれる。本システムの大半は、
*ランタイムが正しくスコープする場所に物を置く* だけのことです。

## 3. 「作業する場所」ではなく「知識が住む場所」で振り分ける

軽い・横断的な作業はどこでやってもよい。山を本当に防ぐ規則は**労働ではなく memory** についてのもの：

- 案件固有の知識 → その**作業員**ディレクトリの memory
- インフラ／参照事実 → **秘書**
- 名簿と横断的な決定 → **リーダー**
- 普遍的な働き方ルール → ランタイムの**グローバル**設定

*どこで動くか* の柔軟さは良い。*知識がどこに落ち着くか* が規律的である限り。

## 4. 安全は「動かないこと」ではなく「戻せること」から来る

「迷ったら何もしない」でシステムを安全にしたくなる。だがそれは目的を捨てる——何も良くならない。
本当の安全は：**先にバックアップ・追加的に変更・1コマンドの undo を持つ**。間違いが安く戻せるから、
明確なケースには*決断的*になれる。可逆性が「動く権利」を買う。

## 5. 確かめる。決めつけない。

「在る/真である/やった」を、そう見えるからと断定しない。実ファイル・実状態・実源を確認してから述べる。
自分の過去の報告も同じ——報告は主張であって検証ではない。

## 6. 最小解釈・良かれの追加をしない

頼まれた事を、最小の忠実なサイズでやる。範囲を広げない・頼まれてない物を「改善」で足さない・
想像上の未来のために先回りで作らない。拡張は別の明示的な決定——既定ではない。

## 7. コスト規律

文脈の肥大は将来の全ターンへの課税（遅く・高く・誤りやすい）。軽くスコープされたワークスペースは
整然としているだけでなく*安くて信頼できる*。死んだ物を削り、全履歴を毎セッション持ち込まない。

## 8. 資産を守る

働き方ルール（蓄積された「我々のやり方」）と既存ツールは**機能であってノイズではない**。
掃除は死荷重を除くもので、良い振る舞いを能動的に導く物を抜いてはいけない。
「ノイズ除去」と「signal の温存」は反対物ではなく、慎重にやる同じ操作です。

## 9. 身の丈で

身の丈に合う分だけ採る。1人運用には少なく、大組織には重い層が要ることもある。
大組織の装置を1人運用に移植しない。付箋でエンタープライズを覆ったふりもしない。
適正サイズにし、何を省いたかを口に出す。
